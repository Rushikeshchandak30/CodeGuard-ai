1. Open this project in VS Code:
   ```powershell
   code C:\Users\rchandak\Downloads\codeguard-ai
   ```


3. Press **F5** (or go to **Run → Start Debugging** → select **"Run Extension"**)

## 6. Run the CLI Locally

The CLI is a standalone Node.js tool — no VS Code required. Use it in terminals, CI, or scripts.

### Quick scan of test-samples
```powershell
cd C:\Users\rchandak\Downloads\codeguard-ai\cli
node dist/index.js scan ..\test-samples
```

### All CLI options
```powershell
# Basic scan (table output)
node dist/index.js scan <project-path>

# JSON output (machine-readable)
node dist/index.js scan <project-path> --format json

# SARIF output (GitHub Code Scanning)
node dist/index.js scan <project-path> --format sarif > results.sarif

# Only report high+ severity
node dist/index.js scan <project-path> --severity high

# Disable specific scanners
node dist/index.js scan <project-path> --no-hallucination --no-vulnerabilities

# Disable MCP scanning
node dist/index.js scan <project-path> --no-mcp

# Pre-commit mode (exits with code 1 if critical/high found)
node dist/index.js pre-commit <project-path>

# Show help
node dist/index.js --help
```

### Install CLI globally (use `codeguard` from anywhere)
```powershell
cd C:\Users\rchandak\Downloads\codeguard-ai\cli
npm link

# Then from any directory:
codeguard scan C:\my-project
codeguard pre-commit C:\my-project
```

## 8. Configure Policy

Create `.codeguard/policy.json` in any project root to govern security rules.

### Option A — Via command
Open the Extension Development Host → **Ctrl+Shift+P** → `CodeGuard: Create Default Policy`

### Option B — Manual

Create `<your-project>/.codeguard/policy.json`:

```json
{
  "version": 1,
  "mode": "warn",
  "rules": {
    "max_critical_findings": 0,
    "max_high_findings": 5,
    "forbidden_packages": ["lodash@<4.17.21", "event-stream"],
    "allowed_packages": ["express", "react"],
    "max_ai_code_ratio": 0.8,
    "require_secrets_scanner": true,
    "require_sast_scanner": true,
    "block_unprovenanced_packages": false,
    "required_provenance_level": "none",
    "max_allowed_severity": "critical",

    "block_npx_mcp_servers": true,
    "block_unencrypted_mcp": true,
    "block_mcp_hardcoded_credentials": true,
    "max_mcp_issues": 0,
    "require_ai_sbom": false,
    "allowed_ai_sdks": ["openai", "anthropic"],

    "forbidden_patterns": [
      {
        "pattern": "TODO:\\s*HACK",
        "message": "Hack TODO found — review before shipping",
        "severity": "warning"
      }
    ]
  }
}
```

### Policy modes

| Mode | Behaviour |
|------|-----------|
| `"audit"` | Log violations to Output Channel only (silent in editor) |
| `"warn"` | Yellow VS Code warnings + CLI warnings (default) |
| `"enforce"` | Red VS Code errors — fails CI pre-commit checks |

## 10. MCP Server Scanner

Automatically scans these files whenever they change:

| File | Tool |
|------|------|
| `.vscode/mcp.json` | VS Code MCP |
| `.cursor/mcp.json` | Cursor IDE |
| `mcp.json` | Generic |
| `claude_desktop_config.json` | Claude Desktop |
| `cline_mcp_settings.json` | Cline |
| `.mcp.json` | Generic |

### What it detects

| Category | Example | Severity |
|----------|---------|----------|
| **Tool Poisoning** | `"ignore previous instructions"` in tool description | Critical |
| **Rug-Pull Risk** | `npx` or `bunx` in command (downloads latest at runtime) | High |
| **Credential Exposure** | Hardcoded `AWS_SECRET_KEY` in env | Critical |
| **Unencrypted Transport** | `http://` remote server URL | High |
| **Pipe-to-Shell** | `curl ... \| bash` | Critical |
| **Prompt Injection** | Hidden Unicode chars in tool description | High |
| **Cross-Origin Escalation** | Tool accessing filesystem outside workspace | High |

### Test it manually

Create `.vscode/mcp.json` in a test project:
```json
{
  "mcpServers": {
    "risky-server": {
      "command": "npx",
      "args": ["-y", "some-mcp-package"]
    },
    "cred-leak": {
      "command": "node",
      "args": ["server.js"],
      "env": {
        "AWS_SECRET_ACCESS_KEY": "AKIAIOSFODNN7EXAMPLE"
      }
    }
  }
}
```

Then run: **Ctrl+Shift+P** → `CodeGuard: Scan MCP Server Configurations`

## 12. CI/CD Integration

### GitHub Actions

Create `.github/workflows/codeguard.yml` in your repo:

```yaml
name: CodeGuard AI Security Scan
on: [push, pull_request]

jobs:
  security-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'

      - name: Install CLI
        run: |
          cd cli
          npm install
          npx tsc -p ./tsconfig.json

      - name: Run Security Scan
        run: |
          node cli/dist/index.js scan . --format sarif > results.sarif
          node cli/dist/index.js scan . --severity high

      - name: Upload SARIF to GitHub Code Scanning
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: results.sarif
```

### Jenkins / GitLab CI

```bash
# In your CI script
cd /path/to/codeguard-ai/cli
node dist/index.js scan $CI_PROJECT_DIR --format json > codeguard-results.json
node dist/index.js pre-commit $CI_PROJECT_DIR  # exits 1 if critical/high found
```

Set in VS Code Settings (**Ctrl+,**) or `settings.json`:

```json
{
  "codeguard.enableHallucinationDetection": true,
  "codeguard.enableSecretsScanner": true,
  "codeguard.enableCodeVulnScanner": true,
  "codeguard.enableSecurityScore": true,
```

|---------|--------|---------|-------------|

## 16. Troubleshooting

### `"Cannot find module 'vscode'"` in CLI
The CLI is vscode-free by design. This error means you're importing from `src/` (which has VS Code deps) instead of `cli/src/`. Always use `cli/dist/index.js`.

### Extension not activating (F5)
1. Run `npm run compile` — ensure no TypeScript errors
2. Check `out/extension.js` exists
3. In the Extension Development Host, open **Help → Toggle Developer Tools → Console** for errors

### `preLaunchTask` fails on F5
The default build task runs `npm run watch`. If it fails, run `npm run compile` manually first, then press F5.

### MCP Scanner not finding configs
MCP configs must be in the **workspace root** or in `.vscode/`, `.cursor/` subdirectories. The scanner uses VS Code's workspace file system — not the CLI.

### CLI `node dist/index.js` — `Cannot find module`
The CLI wasn't compiled. Run:
```powershell
cd cli
npx tsc -p ./tsconfig.json
```

### Lint warnings about `eval()`
These are false positives — the scanner contains `\beval\b` as a **detection regex**, not actual `eval()` usage. They are ignored by design.

### TypeScript version warning from eslint
```
WARNING: You are currently running a version of TypeScript which is not officially supported...
```
This is expected — we use TypeScript 5.9 but `@typescript-eslint` officially supports up to 5.3. It still works correctly.
